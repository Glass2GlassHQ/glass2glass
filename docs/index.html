Pillars

Four non-negotiable design pillars.

The architecture optimizes for a single metric: glass-to-glass latency — the time between photon capture and hardware presentation. Every design decision serves that target.

01

Asynchronous execution

Every element is a cooperative Future. No internal OS thread management; runtime-agnostic. Tokio on servers, Embassy on RTOS, wasm-bindgen-futures in the browser.

02

Hardware-first & zero-copy

Buffers stay in DMABUF, Vulkan, CUDA, D3D11, or WebGPU memory domains. CPU memory copies are treated as system faults, not as the default.

03

no_std, alloc-optional, sans-IO

The same pipeline shape runs on a bare-metal Cortex-M with no heap, an RTOS (Embassy / FreeRTOS / Zephyr), a multi-threaded server, a GPU-resident pipeline, or wasm32. alloc is an optional feature; the no-alloc build links no allocator at all. Network and protocol parsers are pure sans-IO.

04

First-class ML

Tensor allocation, reshaping, and pipeline batching are part of graph orchestration. ML elements negotiate Caps::Tensor the same way video elements negotiate Caps::RawVideo.

Workspace

Ten crates. Clean boundaries.

A cargo workspace, MSRV 1.75, edition 2021. OS-coupled elements live behind cargo features so the baseline compiles for bare-metal targets. A C ABI and Python bindings drive the same graphs from any language.

g2g-core
no_std, alloc optional

Traits (AsyncElement, SourceLoop), the static element model, Frame/PipelinePacket, Caps algebra, buffer pools, the pipeline clock, the CSP solver, and the runner. alloc (with the dynamic graph + solver + tooling) is an optional feature. LGPL v2.1+.

g2g-mcu
no_std, no alloc

Heap-free MCU peripheral elements over embedded-hal trait seams (host-tested against the datasheet with mocks): SpiDisplaySink (ST7789 / ILI9341, whole-frame or banded streaming for large panels), GrabberSrc (DCMI/CSI camera), PcmSink (I2S/SAI), the fixed-point G.711 / IMA ADPCM codecs, the hardware HwJpegDec / HwH264Enc codec seams, YuyvToI420 (camera-4:2:2 → encoder-4:2:0 convert), and RtpSink. LGPL v2.1+.

g2g-mcugen
std, host tool

Host graph compiler: a declarative MCU graph (YAML/JSON, audio or video/display) compiles to a monomorphized static pipeline (heap-free Rust) with every ring sized from the graph's frame geometry. Develop on Linux, ship a bounded static build to the MCU. LGPL v2.1+.

g2g-plugin
no_std + alloc

SDK for dynamically loadable plugins: the declare_plugin! macro + an ABI version tag, loaded at runtime via --plugin / $G2G_PLUGIN_PATH. LGPL v2.1+.

🔋

Heap-free MCU pipeline

alloc is optional: the no-alloc build links no allocator and is proven panic-free, with a budgeted ~KB footprint. A static element model (unboxed async, no dyn/Box) over const-generic lend rings. App code needs zero unsafe. Real interrupt/DMA-driven capture: a DMA-completion ISR fills a lock-free SpscFrameRing while the pipeline drains it on wfi, proven on a SysTick interrupt on emulated Cortex-M (lossless, in order, bounded back-pressure). A fault-recovery supervisor turns a returned peripheral fault into a bounded retry / degrade / reset / escalate action, re-arms the faulting stage through a Recover seam, and pets a hardware watchdog only on real progress, so a wedged pipeline resets the chip. The receive direction mirrors the sender: RtpSrc parses RTP (a bounds-checked header parser shared with the std depayloader) and a heap-free JitterBuffer absorbs arrival jitter and reorders by sequence, handling reorder / duplicate / late / loss countably, proven on Cortex-M by reconstructing an ordered PCM stream from a reordered wire. Beyond media, the catalog reaches the I2C sensor and UART control plane: a real SHT3x driver (datasheet command, CRC-8, conversion) feeds a UART telemetry egress, proven on Cortex-M. For the safety / cert market a checkable safety case ships too: a requirements traceability matrix whose every cited proof CI verifies still exists, a conditions-of-use safety manual, and a one-command qualification kit.

Portability

One pipeline. MCU → RTOS → CPU → GPU → WASM.

Because the processing loop needs only core (alloc is optional), the target changes purely by the deployment shell (which executor, which hardware elements). The graph you write is identical on all five — proven, not asserted: one detection-overlay pipeline whose processing stages come from a single shared definition, reused verbatim by the native (CPU) runner and the browser (WASM) build, with reproducible evidence per target in PORTABILITY.md. A sixth is a compile-verified spike: QNX, the safety-certified RTOS.

🔌

MCU (bare-metal, heap-free)

alloc is optional: the no-alloc build links no allocator, is proven panic-free, and has a budgeted ~KB footprint (ROM / static RAM / worst-case stack). A static element model over const-generic lend rings; g2g-mcu peripheral elements over embedded-hal. Runs on QEMU Cortex-M in CI.

⏱️

RTOS (real-time)

The same static pipeline runs bit-exact under bare-metal, Embassy, FreeRTOS, and Zephyr (a drop-in Zephyr module). Stack channels (embassy-sync), no heap churn. Features embassy / embassy-link.

☁️

CPU (server & edge)

Tokio, multi-threaded on servers or current-thread on edge. The full element + protocol stack; per-OS hardware decode (Linux NVDEC/VAAPI, Windows MF, macOS VideoToolbox, Android MediaCodec). Pair with OrtInference for ML.

⚡

GPU (zero-copy)

Frames stay in Vulkan / CUDA / wgpu / DMABUF domains: Vulkan Video decode → wgpu::Texture, native NVDEC / NVENC, a CUDA ↔ wgpu bridge — no PCIe round-trip.

🌐

WASM (browser)

wasm32, single-threaded (no cross-origin isolation). WebSocket / WebRTC ingest, WebCodecs decode, Canvas / WebGPU present, in-browser or server-offloaded ML.

🛡️

QNX (safety-cert RTOS)

The reference RTOS for ISO 26262 / IEC 62304 automotive & medical, and the std-capable path (aarch64 / x86-64), not the MCU one. A Tier-0 portability spike, compile-checked, not yet run: g2g-core (no-alloc and full alloc + runtime), g2g-mcu, and the g2g-plugins no_std baseline compile for nto-qnx800 with zero code changes — Linux HW is excluded by target_os gating, not pulled in.

🎯

One API across all five

Same AsyncElement trait, same Caps, same negotiation, same runner primitives. Move a stage between browser, edge, and server by swapping one element.

Quickstart

Six lines from clone to live pixels.

Pick a pipeline shape, enable the right cargo features, and run. Detailed smoke-test recipes (RTSP feed, NVDEC, KMS, VAAPI, ML) in the README. 

# Cargo.toml
[dependencies]
g2g-core    = { git = "https://github.com/Glass2GlassHQ/glass2glass" }
g2g-plugins = { git = "https://github.com/Glass2GlassHQ/glass2glass", features = ["rtsp", "ffmpeg", "wayland-sink"] }

// main.rs
use g2g_core::runtime::{run_source_transform_sink, LatencyProfile};
use g2g_plugins::{RtspSrc, ffmpegdec::{FfmpegH264Dec, OutputFormat}, WaylandSink};

let src  = RtspSrc::new("rtsp://localhost:8554/pattern");
let dec  = FfmpegH264Dec::new().with_output_format(OutputFormat::Nv12);
let sink = WaylandSink::new();

run_source_transform_sink(src, dec, sink, &clock, LatencyProfile::Live).await?;
# Linux + NVIDIA; zero PCIe round-trip after decode.
[dependencies]
g2g-plugins = { git = "...", features = ["rtsp", "ffmpeg", "cuda", "cuda-gl"] }

use g2g_plugins::{RtspSrc, ffmpegdec::{FfmpegH264Dec, Backend}, CudaGlSink};

let src  = RtspSrc::new(url);
let dec  = FfmpegH264Dec::with_backend(Backend::NvdecCuda);  // MemoryDomain::Cuda
let sink = CudaGlSink::new();                          // EGL + NV12 shader

run_source_transform_sink(src, dec, sink, &clock, LatencyProfile::Live).await?;
# RTSP -> decode -> GPU preprocess -> ORT (CUDA EP) -> classification.
[dependencies]
g2g-plugins = { git = "...", features = ["rtsp", "ffmpeg"] }
g2g-ml      = { git = "...", features = ["wgpu", "cuda"] }

use g2g_core::runtime::run_linear_chain;
use g2g_ml::{WgpuPreprocess, OrtInference, TensorPostprocess};

let src       = RtspSrc::new(url);
let mut dec   = FfmpegH264Dec::new().with_output_format(OutputFormat::Nv12);
let mut pre   = WgpuPreprocess::new(w, h);                       // NV12 -> f32 NCHW
let mut infer = OrtInference::from_memory_with_cuda(model)?;
let mut post  = TensorPostprocess::topk_classification(5);

run_linear_chain(src, vec![&mut dec, &mut pre, &mut infer, &mut post],
                 FakeSink::new(), &clock, LatencyProfile::Live).await?;
# Loopback-verified; honors receive-side NACK by retransmit.
[dependencies]
g2g-plugins = { git = "...", features = ["udp-egress", "mf-encode"] }

use g2g_plugins::{VideoTestSrc, MfEncode, UdpSink};

let src  = VideoTestSrc::new(1920, 1080, 30, 0);              // RGBA test pattern
let enc  = MfEncode::new_low_latency();                       // Windows; on Linux use the bridge
let sink = UdpSink::new("239.0.0.1:5004".parse()?)
            .with_rtp(96, 0x1234_5678);                     // payload type, SSRC

run_source_transform_sink(src, enc, sink, &clock, LatencyProfile::Live).await?;
# wasm32-unknown-unknown; needs --cfg=web_sys_unstable_apis for WebCodecs.
[dependencies]
g2g-plugins = { git = "...", features = ["web", "web-codecs"] }

use g2g_plugins::{WebSocketSrc, H264Parse, WebCodecsDecode, CanvasSink};

// Browser glass-to-glass: WS bytes -> parse -> WebCodecs -> canvas.
let src    = WebSocketSrc::connect("wss://feed.example.com/h264").await?;
let parse  = H264Parse::new();
let decode = WebCodecsDecode::new();
let sink   = CanvasSink::by_id("video-canvas");

wasm_bindgen_futures::spawn_local(async move {
    run_linear_chain(src, vec![&mut parse, &mut decode], sink, &clock, profile)
        .await.unwrap();
});
# A declarative graph (JSON/YAML) or a Rhai builder script -> the same Graph.
g2g-launch --graph pipeline.yaml
g2g-launch --script build.rhai

# scriptrouter: route each buffer to your own code, per decision. route()
# returns one port, a negative to drop, or an array of ports to multicast.
g2g-launch videotestsrc num-buffers=8 ! \
    scriptrouter name=r script="fn route(f){ f.sequence % 2 }" \
    r. ! appsink channel=even   r. ! appsink channel=odd

// Pull each channel live (Rust / C / Python) while the pipeline runs:
let even = register_appsink_pull("even");
while let Some(frame) = even.pull().await { /* feed your consumer */ }

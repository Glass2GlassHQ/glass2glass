Pillars

Four non-negotiable design pillars.

The architecture optimizes for a single metric: glass-to-glass latency — the time between photon capture and hardware presentation. Every design decision serves that target.

01

Asynchronous execution

Every element is a cooperative Future. No internal OS thread management; runtime-agnostic. Tokio on servers, Embassy on RTOS, wasm-bindgen-futures in the browser.

02

Hardware-first & zero-copy

Buffers stay in DMABUF, Vulkan, CUDA, D3D11, or WebGPU memory domains. CPU memory copies are treated as system faults, not as the default.

03

no_std + alloc, sans-IO

The same pipeline shape runs on a Cortex-M, a multi-threaded server, or wasm32. Network and protocol parsers are pure sans-IO, with I/O stripped out of the logic layer.

04

First-class ML

Tensor allocation, reshaping, and pipeline batching are part of graph orchestration. ML elements negotiate Caps::Tensor the same way video elements negotiate Caps::RawVideo.

Workspace

Nine crates. Clean boundaries.

A cargo workspace, MSRV 1.75, edition 2021. OS-coupled elements live behind cargo features so the baseline compiles for bare-metal targets. A C ABI and Python bindings drive the same graphs from any language.

g2g-core
no_std + alloc

Traits (AsyncElement, SourceLoop), Frame/PipelinePacket, Caps algebra, buffer pools, the pipeline clock, the CSP solver, and the runner. LGPL v2.1+.

g2g-plugin
no_std + alloc

SDK for dynamically loadable plugins: the declare_plugin! macro + an ABI version tag, loaded at runtime via --plugin / $G2G_PLUGIN_PATH. LGPL v2.1+.

g2g-plugins
mixed

The element library: RTSP / RTP / HTTP / HLS / DASH / RTMP ingest, V4L2 / PipeWire / MF capture, ffmpeg (H.264/265, VP8/9, AV1) / VAAPI / MF / VideoToolbox (macOS) / MediaCodec (Android) decoders, AV1 / VP8/9 / MJPEG encoders, container mux/demux (MP4, MPEG-TS, Matroska/WebM, FLV, Ogg), Wayland / KMS / D3D11 / wgpu sinks, ALSA / PulseAudio / PipeWire / WASAPI audio, Opus, compositor, the gst-launch DSL, Embassy, web. LGPL v2.1+.

g2g-ml
std

ONNX Runtime inference (OrtInference) with CUDA + DirectML EPs, pure-Rust Burn inference (BurnInference) on wgpu, the keep-on-GPU wgpu branch (WgpuPreprocess + WgpuInference, GPU-resident tensors end to end), classification head. LGPL v2.1+.

g2g-bridge
std, cdylib

C-FFI dynamic library to embed g2g sub-graphs inside legacy GStreamer pipelines. libgstglass2glass.so. LGPL v2.1+.

g2g-enterprise
std

Lock-free multi-channel tensor batcher that fans N camera streams into a single GPU tensor execution array. AGPL v3.

g2g-python
std

Hosts gst-python-ml elements as first-class g2g elements via embedded CPython (pyo3): transform / source / aggregator, zero-copy frames, GIL offload, analytics metadata. LGPL v2.1+.

g2g-capi
std, cdylib

C ABI (include/g2g.h) to drive whole pipelines from any language: parse_launch + run + bus poll, plus appsrc / appsink buffer I/O with a zero-copy lend. LGPL v2.1+.

g2g-pyapi
std, pyo3

Python bindings (the inverse of g2g-python): g2g.Pipeline(desc) runs a graph on a background thread, AppSrc.push / AppSink.pull feed and drain it, bus_poll watches it. LGPL v2.1+.

designed for replacement
vs GStreamer

Statically-typed graphs replace runtime string-keyed plugin factories. CSP caps negotiation with structured failures. Per-link memory-domain policies.

Capabilities

Built for production multimedia.

Receive, decode, transform, infer, encode, and egress — with the memory-domain handoffs you'd expect from a hardware-first stack.

🎥

Receive: RTSP / RTP / HTTP / HLS / DASH / RTMP / WebRTC / V4L2

RtspSrc over retina, UdpSrc raw-RTP ingest, HttpSrc byte streams, HlsSrc / DashSrc adaptive streaming (live + AES / SAMPLE-AES), RtmpSrc publisher ingest, WebRtcWhepSrc WHEP ingest (str0m), V4l2Src capture, WebSocketSrc / data-channel WebRtcSrc (browser), FileSrc, Mp4Src.

Core
🔌

Decode + encode: ffmpeg / VAAPI / MF / VideoToolbox / MediaCodec / WebCodecs

FfmpegVideoDec (H.264/265, VP8/9, AV1) with sw / NvdecCuvid / NvdecCuda / VAAPI backends (the VAAPI hwaccel is the Linux AMD / Intel path, ffmpegvaapidec); MfDecode (Windows), VtDecode (macOS VideoToolbox), MediaCodecDec (Android NDK, H.264 / H.265, on-device validated, with a zero-copy GPU-texture output path), WebCodecsDecode; encoders Av1Enc (rav1e), VpxEnc (libvpx), MjpegDec / MjpegEnc (pure Rust), VtEncode (macOS), MfEncode (Windows).

GPU
📂

Display: Wayland / KMS / D3D11 / CUDA-GL

WaylandSink for desktop, KmsSink for tty / production, D3D11Sink on Windows, CudaGlSink for zero-copy NVDEC presentation.

GPU
🧠

Inference: ORT + Burn + keep-on-GPU wgpu branch

OrtInference with CUDA + DirectML EPs, pure-Rust BurnInference on wgpu, TensorBatcher for N-stream fan-in. The keep-on-GPU branch (M213–217): WgpuPreprocess imports a GPU NV12 texture and emits a GPU-resident tensor, WgpuInference binds it directly — pixels never leave the GPU between preprocess and infer; zero-copy frame fan-out through a tee. On Android the same branch consumes the RGBA texture MediaCodecDec emits, so decode → preprocess → infer all stay on the GPU.

ML
📢

Egress: RTP / UDP + WebRTC (WHIP)

Sans-IO RtpH264Packetizer (RFC 3550 + RFC 6184) and a tokio-driven UdpSink with 90 kHz PTS mapping, honoring receive-side NACK from a bounded send history. WebRtcSink publishes H.264 to a WHIP server over a str0m PeerConnection (ICE / DTLS / SRTP, pure-Rust crypto) — g2g owns the socket, str0m owns the protocol.

Core
📡

RTP resilience: jitter buffer + RTCP + NACK

Sans-IO RtpJitterBuffer (reorder, bounded-latency loss handling, dup drop) and rtcp (RFC 3550 SR/RR + RFC 4585 Generic NACK, reception stats). UdpSrc reports + NACKs; the loop recovers loss end to end.

Core
🎬

Compositor: pixel mixer

Compositor overlays N raw RGBA8 inputs onto one canvas (position, z-order, per-pad alpha, bilinear scale, background colour) for picture-in-picture, camera grids, and sub-window UIs — input-0-driven cadence, CPU / no_std baseline.

Core
📦

Containers: MP4 / TS / MKV / FLV / Ogg

Pure-Rust demux + mux: mp4src / mp4sink (fMP4), tsdemux / mpegtsmux, matroskademux / matroskamux (WebM), flvdemux / flvmux, oggdemux, fmp4demux (CMAF) — on the typed Caps::ByteStream link with typefind sniffing and a tag system on the bus.

Core
📺

Adaptive streaming: HLS / DASH / RTMP

HttpSrc byte-stream fetch under HlsSrc (TS + fMP4/CMAF, live reload, AES-128 + SAMPLE-AES decrypt) and DashSrc (SegmentTemplate / SegmentTimeline, $Number$/$Time$, dynamic MPDs). RtmpSrc accepts an OBS / ffmpeg publisher over a sans-IO RTMP session.

Core
📑

Parsers + transforms

Bitstream parsers h264parse / h265parse / aacparse / opusparse / vp8parse / vp9parse / av1parse feed negotiation; SW transforms videoscale / videorate / videocrop / videoflip / videobalance / videobox / alpha and audioconvert / audioresample / audiomixer / volume.

Core
📝

gst-launch DSL + introspection

parse_launch("a key=v ! caps ! tee ! ...") builds a runnable Graph from text against a Registry: properties (set_property / get_property), caps-filter grammar, tee branching, and muxer fan-in. inspect(name) is the gst-inspect analog.

Core
🧩

Language bindings: C ABI + Python

The launch DSL is the cross-language waist. g2g-capi exposes a C ABI (g2g.h) to parse + run a pipeline and poll its bus; g2g-pyapi wraps it for Python. appsrc / appsink feed and drain frames across the boundary, with a zero-copy buffer lend so an app pointer reaches the graph without a copy.

Core
🎯

Per-frame metadata + analytics overlay

Typed FrameMeta / AnalyticsMeta relation graph (detection / classification / tracking) survives tee fan-out copy-on-write. DetectionPostprocess (YOLOv8 + NMS) produces it; AnalyticsOverlay (CPU) and VelloAnalyticsOverlay → WgpuSink (GPU, no readback) draw it.

ML
🎼

Audio: ALSA / PulseAudio / PipeWire / WASAPI + Opus + AAC

Linux out via AlsaSink / PulseSink / PipeWireSink (+ PipeWireSrc capture), WasapiSink / WasapiSrc on Windows (incl. loopback); OpusEnc / OpusDec (libopus), MfAacEncode / MfAacDecode via Media Foundation, Mp4AudioSink in fMP4, audiomixer / audioresample.

Core
🔗

CSP caps negotiation

CapsSet with alternatives + preferences, solve_linear with arc consistency, structured NegotiationFailure, mid-stream re-solve over the downstream subgraph (Caps-α).

Core
⚡

Mid-stream allocation cascade

β N-hop allocation re-cascade through a coordinator task: a sink-side allocation proposal reaches the source on a mid-stream CapsChanged via the select2 + per-arm control channel.

Core
📚

Multi-element runner

run_linear_chain(src, Vec<Transform>, sink) drives any-length linear chains. Pad templates as type-level metadata for pre-instantiation compatibility queries.

Core
🏔

Fan-out / fan-in

Router, Gate, Merger primitives; BranchSlot with three SwapPolicys (immediate, drain-old, shadow-warm); tee-to-display+recording with per-branch re-solve.

Core
🛡

Dynamic graph reconfiguration

ElementSlot via arc_swap: hot-swap elements with no pipeline stall. Use for ReFixate, codec change at keyframe, bitrate switch at next segment.

Core
📷

Live clock + sync

Single pipeline reference clock; sources map capture clocks to it; backpressure-paced sync sinks with capture_ns preserved end-to-end for honest latency measurement.

Core
🧙️‍♂️

Embedded / RTOS / Embassy

StaticBufferPool for no-alloc loops, EmbassyClock over embassy-time, PacketChannel zero-alloc inter-task link. Compiles for thumbv7em-none-eabihf + riscv32.

RTOS
🌐

Browser / Wasm

WasmClock, WebSocketSrc, WebCodecsDecode, CanvasSink. First in-browser glass-to-glass pipeline: WS → WebCodecs → Canvas.

Web
🔥

CUDA zero-copy track

MemoryDomain::Cuda in core, Backend::NvdecCuda keeps NV12 in device memory, CudaDownload + CudaGlSink via CUDA↔GL interop (not dmabuf — doesn't work on NVIDIA).

GPU
📱

Android zero-copy decode → GPU

MediaCodecDec::with_gpu_output() imports the decoded AHardwareBuffer into Vulkan and converts it to an RGBA wgpu::Texture through an immutable VkSamplerYcbcrConversion compute pass (which wgpu's bind-group API cannot express), emitted as MemoryDomain::WgpuTexture for WgpuPreprocess — the decoded frame never touches the CPU. Validated on a Pixel 10a.

GPU
Deployment

One core. Five deployment profiles.

Because the processing loop requires only core and alloc, profiles vary purely by the top-level orchestration binary.

☁️

Enterprise server (cloud)

Tokio multi-threaded runtime. Bounded MPMC flume channels. Hardware: VAAPI / NVDEC / V4L2 with OwnedDmaBuf handles. Features multi-thread, std.

💻

Desktop, mobile & edge

Tokio current-thread. Per-OS hardware decode: Linux FfmpegH264Dec sw / NVDEC + VAAPI, Windows MfDecode + D3D11Sink, macOS VtDecode (VideoToolbox), Android MediaCodecDec (NDK, with a zero-copy decode → GPU → preprocess path). Present via WaylandSink / KmsSink. Pair with OrtInference for on-device ML.

🪫

Bare-metal / RTOS

Embassy executor (single-threaded, cooperative). Stack channels (embassy-sync). DMA-ring capture (planned). No-alloc strict variant via StaticBufferPool.

🌐

Browser sandbox

Web Workers via wasm-bindgen-futures. WebSocket / WebRTC ingest, WebCodecs decode, Canvas / WebGPU output. std feature on wasm32-unknown-unknown.

🔗

GStreamer interop

libgstglass2glass.so bridge lets a g2g sub-graph (wgpu filter, Burn ML) run inside a legacy C GStreamer pipeline. GstDmaBufMemory → OwnedDmaBuf no-op close.

🎯

Single API across all five

Same AsyncElement trait, same Caps, same negotiation, same runner primitives. The deployment shell picks the executor and links the OS-coupled elements.

Quickstart

Six lines from clone to live pixels.

Pick a pipeline shape, enable the right cargo features, and run. Detailed smoke-test recipes (RTSP feed, NVDEC, KMS, VAAPI, ML) in the README. 

# Cargo.toml
[dependencies]
g2g-core    = { git = "https://github.com/Glass2GlassHQ/glass2glass" }
g2g-plugins = { git = "https://github.com/Glass2GlassHQ/glass2glass", features = ["rtsp", "ffmpeg", "wayland-sink"] }

// main.rs
use g2g_core::runtime::{run_source_transform_sink, LatencyProfile};
use g2g_plugins::{RtspSrc, ffmpegdec::{FfmpegH264Dec, OutputFormat}, WaylandSink};

let src  = RtspSrc::new("rtsp://localhost:8554/pattern");
let dec  = FfmpegH264Dec::new().with_output_format(OutputFormat::Nv12);
let sink = WaylandSink::new();

run_source_transform_sink(src, dec, sink, &clock, LatencyProfile::Live).await?;
# Linux + NVIDIA; zero PCIe round-trip after decode.
[dependencies]
g2g-plugins = { git = "...", features = ["rtsp", "ffmpeg", "cuda", "cuda-gl"] }

use g2g_plugins::{RtspSrc, ffmpegdec::{FfmpegH264Dec, Backend}, CudaGlSink};

let src  = RtspSrc::new(url);
let dec  = FfmpegH264Dec::with_backend(Backend::NvdecCuda);  // MemoryDomain::Cuda
let sink = CudaGlSink::new();                          // EGL + NV12 shader

run_source_transform_sink(src, dec, sink, &clock, LatencyProfile::Live).await?;
# RTSP -> decode -> GPU preprocess -> ORT (CUDA EP) -> classification.
[dependencies]
g2g-plugins = { git = "...", features = ["rtsp", "ffmpeg"] }
g2g-ml      = { git = "...", features = ["wgpu", "cuda"] }

use g2g_core::runtime::run_linear_chain;
use g2g_ml::{WgpuPreprocess, OrtInference, TensorPostprocess};

let src       = RtspSrc::new(url);
let mut dec   = FfmpegH264Dec::new().with_output_format(OutputFormat::Nv12);
let mut pre   = WgpuPreprocess::new(w, h);                       // NV12 -> f32 NCHW
let mut infer = OrtInference::from_memory_with_cuda(model)?;
let mut post  = TensorPostprocess::topk_classification(5);

run_linear_chain(src, vec![&mut dec, &mut pre, &mut infer, &mut post],
                 FakeSink::new(), &clock, LatencyProfile::Live).await?;
# Loopback-verified; honors receive-side NACK by retransmit.
[dependencies]
g2g-plugins = { git = "...", features = ["udp-egress", "mf-encode"] }

use g2g_plugins::{VideoTestSrc, MfEncode, UdpSink};

let src  = VideoTestSrc::new(1920, 1080, 30, 0);              // RGBA test pattern
let enc  = MfEncode::new_low_latency();                       // Windows; on Linux use the bridge
let sink = UdpSink::new("239.0.0.1:5004".parse()?)
            .with_rtp(96, 0x1234_5678);                     // payload type, SSRC

run_source_transform_sink(src, enc, sink, &clock, LatencyProfile::Live).await?;
# wasm32-unknown-unknown; needs --cfg=web_sys_unstable_apis for WebCodecs.
[dependencies]
g2g-plugins = { git = "...", features = ["web", "web-codecs"] }

use g2g_plugins::{WebSocketSrc, H264Parse, WebCodecsDecode, CanvasSink};

// Browser glass-to-glass: WS bytes -> parse -> WebCodecs -> canvas.
let src    = WebSocketSrc::connect("wss://feed.example.com/h264").await?;
let parse  = H264Parse::new();
let decode = WebCodecsDecode::new();
let sink   = CanvasSink::by_id("video-canvas");

wasm_bindgen_futures::spawn_local(async move {
    run_linear_chain(src, vec![&mut parse, &mut decode], sink, &clock, profile)
        .await.unwrap();
});

Pillars

Four non-negotiable design pillars.

The architecture optimizes for a single metric: glass-to-glass latency — the time between photon capture and hardware presentation. Every design decision serves that target.

01

Asynchronous execution

Every element is a cooperative Future. No internal OS thread management; runtime-agnostic. Tokio on servers, Embassy on RTOS, wasm-bindgen-futures in the browser.

02

Hardware-first & zero-copy

Buffers stay in DMABUF, Vulkan, CUDA, D3D11, or WebGPU memory domains. CPU memory copies are treated as system faults, not as the default.

03

no_std + alloc, sans-IO

The same pipeline shape runs on a Cortex-M, a multi-threaded server, or wasm32. Network and protocol parsers are pure sans-IO, with I/O stripped out of the logic layer.

04

First-class ML

Tensor allocation, reshaping, and pipeline batching are part of graph orchestration. ML elements negotiate Caps::Tensor the same way video elements negotiate Caps::RawVideo.

Workspace

Five crates. Clean boundaries.

A cargo workspace, MSRV 1.75, edition 2021. OS-coupled elements live behind cargo features so the baseline compiles for bare-metal targets.

g2g-core
no_std + alloc

Traits (AsyncElement, SourceLoop), Frame/PipelinePacket, Caps algebra, buffer pools, the pipeline clock, the CSP solver, and the runner. LGPL v2.1+.

g2g-plugins
mixed

The element library: RTSP, RTP, H.264 parser/AVCC/Annex-B, ffmpeg / VAAPI / MF decoders, Wayland / KMS / D3D11 sinks, WASAPI, fMP4, file I/O, Embassy bindings, web elements. LGPL v2.1+.

g2g-ml
std

ONNX Runtime inference (OrtInference) with CUDA + DirectML EPs, pure-Rust Burn inference (BurnInference) on wgpu, GPU tensor preprocess (WgpuPreprocess), classification head. LGPL v2.1+.

g2g-bridge
std, cdylib

C-FFI dynamic library to embed g2g sub-graphs inside legacy GStreamer pipelines. libgstglass2glass.so. LGPL v2.1+.

g2g-enterprise
std

Lock-free multi-channel tensor batcher that fans N camera streams into a single GPU tensor execution array. AGPL v3.

designed for replacement
vs GStreamer

Statically-typed graphs replace runtime string-keyed plugin factories. CSP caps negotiation with structured failures. Per-link memory-domain policies. ~95% feature parity with GStreamer's core runtime — with the latency edge.

Deployment

One core. Five deployment profiles.

Because the processing loop requires only core and alloc, profiles vary purely by the top-level orchestration binary.

☁️

Enterprise server (cloud)

Tokio multi-threaded runtime. Bounded MPMC flume channels. Hardware: VAAPI / NVDEC / V4L2 with OwnedDmaBuf handles. Features multi-thread, std.

💻

Desktop & edge

Tokio current-thread. Hardware: FfmpegH264Dec sw / NVDEC, WaylandSink / KmsSink / D3D11Sink. Pair with OrtInference for on-device ML.

🪫

Bare-metal / RTOS

Embassy executor (single-threaded, cooperative). Stack channels (embassy-sync). DMA-ring capture (M48, planned). No-alloc strict variant via StaticBufferPool.

🌐

Browser sandbox

Web Workers via wasm-bindgen-futures. WebSocket / WebRTC ingest, WebCodecs decode, Canvas / WebGPU output. std feature on wasm32-unknown-unknown.

🔗

GStreamer interop

libgstglass2glass.so bridge lets a g2g sub-graph (wgpu filter, Burn ML) run inside a legacy C GStreamer pipeline. GstDmaBufMemory → OwnedDmaBuf no-op close.

🎯

Single API across all five

Same AsyncElement trait, same Caps, same negotiation, same runner primitives. The deployment shell picks the executor and links the OS-coupled elements.

Quickstart

Six lines from clone to live pixels.

Pick a pipeline shape, enable the right cargo features, and run. Detailed Fedora + CUDA + Rembrandt iGPU recipes in the README. 

# Cargo.toml
[dependencies]
g2g-core    = { git = "https://github.com/boxerab/glass2glass" }
g2g-plugins = { git = "https://github.com/boxerab/glass2glass", features = ["rtsp", "ffmpeg", "wayland-sink"] }

// main.rs
use g2g_core::runtime::{run_source_transform_sink, LatencyProfile};
use g2g_plugins::{RtspSrc, ffmpegdec::{FfmpegH264Dec, OutputFormat}, WaylandSink};

let src  = RtspSrc::new("rtsp://localhost:8554/pattern");
let dec  = FfmpegH264Dec::new().with_output_format(OutputFormat::Nv12);
let sink = WaylandSink::new();

run_source_transform_sink(src, dec, sink, &clock, LatencyProfile::Live).await?;
# Linux + NVIDIA; zero PCIe round-trip after decode.
[dependencies]
g2g-plugins = { git = "...", features = ["rtsp", "ffmpeg", "cuda", "cuda-gl"] }

use g2g_plugins::{RtspSrc, ffmpegdec::{FfmpegH264Dec, Backend}, CudaGlSink};

let src  = RtspSrc::new(url);
let dec  = FfmpegH264Dec::with_backend(Backend::NvdecCuda);  // MemoryDomain::Cuda
let sink = CudaGlSink::new();                          // EGL + NV12 shader

run_source_transform_sink(src, dec, sink, &clock, LatencyProfile::Live).await?;
# RTSP -> decode -> GPU preprocess -> ORT (CUDA EP) -> classification.
[dependencies]
g2g-plugins = { git = "...", features = ["rtsp", "ffmpeg"] }
g2g-ml      = { git = "...", features = ["wgpu", "cuda"] }

use g2g_core::runtime::run_linear_chain;
use g2g_ml::{WgpuPreprocess, OrtInference, TensorPostprocess};

let src       = RtspSrc::new(url);
let mut dec   = FfmpegH264Dec::new().with_output_format(OutputFormat::Nv12);
let mut pre   = WgpuPreprocess::new(w, h);                       // NV12 -> f32 NCHW
let mut infer = OrtInference::from_memory_with_cuda(model)?;
let mut post  = TensorPostprocess::topk_classification(5);

run_linear_chain(src, vec![&mut dec, &mut pre, &mut infer, &mut post],
                 FakeSink::new(), &clock, LatencyProfile::Live).await?;
# Loopback-verified; RTCP + RTSP ANNOUNCE on the roadmap.
[dependencies]
g2g-plugins = { git = "...", features = ["udp-egress", "mf-encode"] }

use g2g_plugins::{VideoTestSrc, MfEncode, UdpSink};

let src  = VideoTestSrc::new(RawVideoFormat::Nv12, 1920, 1080, 30.0);
let enc  = MfEncode::new_low_latency();                          // no B-frames
let sink = UdpSink::bind("0.0.0.0:0")?
            .with_remote("239.0.0.1:5004")
            .with_rtp(96, 0x12345678);                       // payload type, SSRC

run_source_transform_sink(src, enc, sink, &clock, LatencyProfile::Live).await?;
# wasm32-unknown-unknown; needs --cfg=web_sys_unstable_apis for WebCodecs.
[dependencies]
g2g-plugins = { git = "...", features = ["web", "web-codecs"] }

use g2g_plugins::{WebSocketSrc, H264Parse, WebCodecsDecode, CanvasSink};

// Browser glass-to-glass: WS bytes -> parse -> WebCodecs -> canvas.
let src    = WebSocketSrc::connect("wss://feed.example.com/h264").await?;
let parse  = H264Parse::new();
let decode = WebCodecsDecode::new();
let sink   = CanvasSink::by_id("video-canvas");

wasm_bindgen_futures::spawn_local(async move {
    run_linear_chain(src, vec![&mut parse, &mut decode], sink, &clock, profile)
        .await.unwrap();
});

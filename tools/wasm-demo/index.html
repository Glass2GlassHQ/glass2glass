glass2glass browser pipeline

WebSocketSrc → WebCodecsDecode → CanvasSink — a g2g graph compiled to WebAssembly, decoding H.264 in the browser via WebCodecs (the browser's own hardware decoder) and presenting to the canvas. Single-threaded; no cross-origin-isolation headers needed.

Canvas 2D copies the decoded frame out to system RGBA and paints it. WebGPU imports the decoded VideoFrame as a GPUExternalTexture and samples it in a render pass — no CPU readback (the browser analog of the native Vulkan Video→wgpu wedge).

Detect modes add a detection stage to the linear chain (decode → detect → overlay → canvas). synthetic box plants a fixed box (no model); real YOLOv8 / ort-web runs a real ONNX YOLOv8n via ONNX Runtime Web in the browser; server-side / remote uses the generic WsWireTransform (the distributed-graph primitive) to ship each decoded frame to the native detect-server (the real g2g OrtInference → DetectionPostprocess chain) over the g2g wire codec and draws the boxes it returns — the SAME graph, inference moved off the browser by swapping one generic element (Architecture B). The real/remote modes need the model + a sample clip: run tools/wasm-demo/get-detection-assets.sh first, serve the models/bus_640.h264 fixture, and for remote also run detect-server (ws://127.0.0.1:8090).

Send (encode → egress)

PatternSrc → WebCodecsEncode → WebSocketSink — generate an animated pattern, encode it to H.264 in the browser (WebCodecs VideoEncoder), and stream it over a WebSocket. Run the ws-recv-server and it saves a playable received.h264.

Send camera swaps the synthetic PatternSrc for WebCameraSrc (getUserMedia + MediaStreamTrackProcessor) — a real webcam through the same encode + egress path. Needs camera permission.
